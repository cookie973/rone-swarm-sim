# 论文/答辩功能总结

## 一、系统定位

本系统是一个面向 6 架无人机的蜂群协同控制系统，运行环境为：

- Ubuntu 22.04
- ROS 2 Humble
- ArduPilot SITL
- Gazebo Harmonic

系统目标是实现多无人机编队协同、任务切换与搜索行为控制，并提供上位机交互界面。

## 二、系统结构

系统由三层组成：

1. 飞控与仿真层
   - Gazebo + ArduPilot SITL

2. 集群控制层
   - `formation_commander`
   - 负责队形切换、任务规划、任务状态管理、搜索控制

3. 上位机交互层
   - `swarm_ground_station`
   - 负责任务下发、快速控制和状态显示

## 三、已经完成的核心功能

### 1. 编队控制功能

已实现：

- 原始编队恢复
- 横队 `line_x`
- 纵队 `line_y`
- V 型编队 `v_shape`
- T 型编队 `t_shape`
- 梯队 `echelon`
- 矩形编队 `rectangle`

支持两类切换场景：

- 静止状态编队切换
- 飞行过程中的编队切换

### 2. 任务控制功能

已实现：

- 编队整体飞往目标点
- 当前点位降落
- 原点返航降落
- 清空当前任务

### 3. 搜索功能

已实现两类搜索模式：

1. 搜寻模式 1：多目标点加权搜索
   - 输入多个目标点
   - 按权重分配无人机数量
   - 自动形成无碰撞搜索分布

2. 搜寻模式 2：单点盘旋搜索
   - 编队飞往目标中心
   - 进入盘旋搜索状态

### 4. 快速控制功能

已实现：

- 一键起飞
- 当前点位起飞
- 当前点位降落
- 返回初始位置降落

### 5. 状态监控功能

上位机可实时显示：

- 当前控制阶段 `phase`
- 当前队形 `formation`
- 当前任务目标 `mission_goal`
- 搜索任务状态
- 盘旋任务状态
- 每架无人机的模式、解锁状态、位置

## 四、关键技术亮点

### 1. 飞行中编队切换优化

为了解决飞行中编队切换效率低、路径冲突多的问题，系统实现了：

- 冲突感知目标分配
- 最小图着色式高度分层避碰
- 重构阶段与任务飞行并行进行

效果：

- 高冲突场景 `line_x -> v_shape` 切换总时长优化到约 `24.5 s`

### 2. 当前状态出发的任务规划

系统支持：

- 新任务直接从当前状态开始规划
- 不必先回原点或回原始编队

这提高了任务切换的连续性和实时性。

### 3. 多目标加权搜索分配

系统支持对多个目标点设置不同权重，控制器可根据权重自动分配无人机数量，实现：

- 重点区域投入更多无人机
- 非重点区域投入更少无人机

### 4. 搜索与任务状态统一管理

系统已实现统一的任务状态清理机制，包括：

- mission 状态
- search 状态
- orbit 状态
- steady target 状态

这解决了人工介入后旧任务残留的问题。

## 五、已完成验证

### 1. 起飞验证

已完成 6 机起飞验证：

- 全部无人机可进入 `GUIDED`
- 高度可稳定到约 `5.0 m`

### 2. 编队飞行验证

已验证：

- 编队整体前往目标点
- 飞行中编队切换
- 静止编队切换

### 3. 搜索模式验证

- 多目标加权搜索
- 单点盘旋搜索

其中多目标搜索已验证权重分配结果，例如：

- 权重 `5.0 / 2.5 / 1.0`
- 最终分配结果约为 `3 / 2 / 1`

### 4. 降落验证

已验证：

- 当前点位降落
- 原点返航降落

## 六、系统现阶段边界

当前系统已经满足演示、联调和论文展示需求，但边界也明确：

1. 当前重点是 SITL 仿真验证，不是实机部署
2. 当前上位机以已有 Tk 界面为主，强调功能完整性而非复杂地图交互
3. 某些流程在地面安全态下会主动清理任务，这是控制安全策略的一部分

## 七、论文中可直接使用的总结表述

本课题已完成一个基于 ROS 2、ArduPilot SITL 与 Gazebo 的六机蜂群协同控制系统，实现了多无人机编队控制、飞行中编队切换、多目标加权搜索、单点盘旋搜索、统一任务管理以及上位机交互控制。系统能够在仿真环境中稳定完成起飞、编队机动、任务切换、搜索执行和降落等完整流程，具备较好的功能完整性与演示价值。

## 八、答辩时建议强调的四点

1. 不是单机控制，而是集群级任务控制
2. 支持飞行中的编队重构，不需要先停下来
3. 支持按权重进行多目标搜索分配
4. 已经完成端到端功能验证，不是停留在算法设计阶段
